# frontend-design-guide

用于当前工作区的前端设计和 UI 质量判断。

## 适用场景

- 新建或改造页面、组件、弹窗、表单、列表、仪表盘。
- 需要判断布局、视觉层级、交互状态和响应式设计。
- 需要把产品需求转成前端界面结构。

## 执行原则

1. 先确认用户目标、主要用户、关键操作和状态。
2. 先复用项目已有组件、样式变量和设计约定。
3. 明确空态、加载态、错误态、禁用态、边界数据。
4. 避免模板化堆砌，保持视觉层级和信息密度。
5. 修改后尽量截图或运行页面验证。
