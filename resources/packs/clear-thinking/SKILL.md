# clear-thinking

当前工作区的思考辅助技能。它不是资料汇编、来源索引或提示词清单，而是一个轻量的“微技能操作系统”：

```text
事实校准 → 处境诊断 → 微技能选择 → 组合推理 → 最小行动 → 验证修正
```

## 1. 用途

负责：

- 判断事实、目标、主要矛盾、下一步行动和验证方式。
- 按问题处境选择最小充分微技能组合。
- 在可行动、需查证、需追问、需子代理或需 Workflow 时停止继续分析。
- 不按来源材料、书名、人物或案例背书；素材只在建设阶段被吸收为判断动作。

## 2. 何时使用

优先用于：

- 用户要求分析、判断、规划、复盘、取舍、找关键问题。
- 用户指出回答“浅”“像提示词”“没提炼方法论”“太教条”。
- 当前问题涉及多目标冲突、长期后果、系统约束，或用户/组织/技术/商业之间的张力。
- 需要先决定“做什么、不做什么、先做哪一步、怎么验收”。

不要用于：

- 单点明确小任务、一次性短答、低风险直接操作。
- 用户明确要求不要分析、只执行。

## 3. 渐进读取

默认只读 `SKILL.md`。只有资源会改变本轮判断、行动或验证方式时，才按最小必要继续读取。

| 任务信号 | 读取文件 | 用途 |
| --- | --- | --- |
| 需要判断问题卡在哪类动作、如何组合、何时停止 | `resources/ROUTER.md` | 路由、组合、停止/升级条件 |
| 已确定动作类型，需要找具体微技能 | `resources/MICRO_SKILLS.md` | 定位候选微技能 |
| 已确定采用某个微技能 | `resources/micro-skills/<微技能>.md` | 执行具体判断动作 |
| 完成前检查，或用户要求“别浅/别教条/检查是否靠谱” | `resources/CHECKLISTS.md` | 输出质量和反误用检查 |

读取数量：

- 默认读取 1-3 个具体微技能文件。
- 复杂任务最多读取 5 个；超过 5 个先裁剪主要矛盾。
- 仍需要 6 个以上时，不在主会话硬读，优先拆子代理或 Workflow。

采用规则：

- 只看 `MICRO_SKILLS.md` 只能称为“候选”。
- 凡在输出中声明“采用”的微技能，必须已读取对应文件，并核对其中的“操作”和“失效信号”。
- 不要为了展示方法论而读取资源文件；不要为了“显得有依据”复述来源材料。

## 4. 执行协议

内部执行，不必完整展示：

```text
1. 感知输入：事实、目标、约束、缺口、风险。
2. 判断是否真的需要 clear-thinking；单点明确任务直接执行。
3. 轻量问题直接用本文件输出当前判断、最小行动和验证方式。
4. 需要路由时读 ROUTER.md；需要定位微技能时读 MICRO_SKILLS.md。
5. 只读取最小充分的具体微技能文件，组合后输出。
6. 达到可行动、需工具、需用户确认、需子代理或需 Workflow 时停止继续分析。
```

复杂问题内部先形成这张卡：

```markdown
- 问题处境：
- 主要矛盾：
- 关键缺口：
- 本轮采用微技能：
- 本轮不用的微技能：
- 组合顺序：
- 停止条件：
- 验证方式：
```

只有当用户要求解释方法时，才展示规划依据。

## 5. 硬规则

- 先事实后框架：事实不足时先查证或定位信息缺口。
- 先主要矛盾后建议清单：只围绕本轮最关键张力展开。
- 先最小行动后深度分析：能行动就停止分析。
- 不按关键词套模板：产品、创业、痛苦、生活、某本书都只是输入信号，不是路由结果。
- 不用书名、人物、公司案例、宏大口号替代当前事实和验证路径。
- 不把增长理解成作弊式拉新；不能牺牲隐私、信任、长期留存和用户尊严。
- 多材料、多阶段或需要交叉验证时，先拆分任务；不要在主会话硬塞长材料。

## 6. 输出门槛

触发本技能后的合格输出至少满足：

1. 有当前判断，不能只列框架。
2. 区分事实、推断、假设和价值偏好。
3. 说明本轮主要矛盾，不平均展开。
4. 给出下一步、验收方式、失败信号和后手。
5. 说明停止点：是否应行动、查证、追问、使用工具、拆子代理或 Workflow。

默认短输出：

```markdown
## 结论
## 依据与假设
## 主要矛盾
## 最小行动与验证
## 风险与边界
```
