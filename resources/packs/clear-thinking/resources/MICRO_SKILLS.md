# clear-thinking 微技能索引

本文件只用于定位微技能文件。凡在输出中声明“采用”的微技能，必须先读取对应 `micro-skills/<微技能>.md` 并核对“操作”和“失效信号”。

默认按 `ROUTER.md` 判断动作类型后，再来本索引查候选；不要为了展示方法论而读取全部微技能。

## 1. 动作类型导航

| 动作类型 | 解决的判断缺口 | 默认产物 |
| --- | --- | --- |
| 事实校准 | 事实、推断、样本、信息缺口不清 | 事实边界、证据质量、待验证信息 |
| 目标取舍 | 目标、代价、优先级和不可兼得项不清 | 目标重写、取舍理由、代价清单 |
| 问题诊断 | 问题很多但不知道卡在哪一层 | 问题类型、主要矛盾、层级诊断 |
| 风险边界 | 高成本、不可逆、危机、责任边界不清 | 风险清单、止损线、切换条件 |
| 结构治理 | 多人、多钱、多权责或机制承接不清 | 权责结构、反馈机制、承接链 |
| 产品表达 | 需要把需求、页面、字段、体验表达清楚 | 需求定义、页面/字段/体验方案 |
| 增长信任 | 用户、口碑、可信度、长期资产不清 | 信任证据、增长机制、复利路径 |
| 执行验证 | 已经有判断但缺行动、验收或复盘 | 最小行动、验收标准、完成判定 |
| 关系承接 | 关系、人情、外部帮助、主体责任不清 | 边界、可拒绝事项、主体承接清单 |
| 方法建设 | 需要从材料中提炼或封装方法 | 判断动作、微技能草案、反背书检查 |

## 2. 微技能索引

字段说明：

- **触发信号**：什么时候考虑它。
- **最小产物**：读完该微技能后应形成的差异化中间结果。
- **标签**：只保留会影响候选排序的场景标签。
- **可选补充**：最多列关键补充，不能自动等同于全都采用。
