# 承接链结构设计

> 动作类型：结构治理
> 场景标签：经营、增长、创业、长期判断、工作、生活、生态合作、帮扶、赋能

## 适用处境

你有资源或能力想长期帮助一个团队、生态伙伴、用户群体或亲友，但直接包办会制造依赖、责任转嫁或不可持续承诺。

## 不适用处境

紧急救助必须先救急；法定、伦理、公共责任不能外包；对方没有基本承接能力；帮助方无法长期兑现承诺；链条依赖一方无偿透支。

## 输入

- 被帮助方能力、外部资源、参与方和权责。
- 交易/反馈机制、质量标准、成本结构、交付机制。
- 合规边界、退出条件、基本保障责任。

## 操作步骤

1. 区分救急、扶持、合作、交易：救急先救，长期结构另设。
2. 写清外部提供什么：资金、渠道、训练、工具、订单、反馈、保护还是背书。
3. 写清对方必须自己承担什么：质量、成本、交付、学习、维护、复盘和风险。
4. 选择承接节点：从实际执行中识别能稳定反馈、组织协作、解决问题的人或机制，并写清替换信号。
5. 设计交易、反馈或协作链条，让质量、成本、交付有自我约束。
6. 明确不能转嫁的法律、伦理、劳动保护、公共责任和基本保障责任。
7. 设定退出条件，避免永久包办或突然抽离造成二次伤害。

## 关键自问

- 我是在帮对方形成能力，还是替对方承担结果？
- 这个链条中谁对质量、成本、交付负责？
- 如果我退出，结构是否立刻崩塌？
- 是否有一方长期无偿透支？
- 哪些责任无论如何不能下放到链条末端？

## 输出格式

```markdown
- 帮助类型：
- 外部资源：
- 对方承接责任：
- 反馈机制：
- 交易/协作机制：
- 不可转嫁责任：
- 依赖风险：
- 退出条件：
```

## 常见误用

- 把包办包装成赋能。
- 把公共责任、劳动保护或基本保障外包给链条末端。
- 只设计资源流，不设计反馈、质量和退出机制。
- 用长期承诺制造对外部资源的依赖。

## 失效信号

对方无承接能力、法律要求直接保障、紧急救助、基本责任不可转嫁、劳动保护或公共责任被外包时不用；当链条稳定依赖某一方无偿透支时，应停止并重构。

## 最高风险边界

不能用“承接链”把不可转嫁的保障、劳动、法律、伦理或公共责任层层下放。

## 常组合

- `相关方损益与承接力地图`：多方承接者、受影响者和风险边界不清时，先画中性地图。
- `样本试点回流循环`：承接链需要先在少量点位验证再推广时。
- `承接责任检查`：确认主体必须自承接的动作。
- `资本控制权校准`：涉及出资、控制权和风险承担时。
- `生态协同`：多方合作需要形成互惠结构时。
- `最小行动`：先做小范围结构验证。
