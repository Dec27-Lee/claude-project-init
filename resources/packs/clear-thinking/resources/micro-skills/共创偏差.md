# 共创偏差

> 动作类型：事实校准
> 场景标签：AI、组织、产品、协作、工作

## 适用处境

共创用户反馈很好、专家认可、试点积极，但真实推广、付费、复用或一线使用是否成立仍不确定。

## 输入

- 反馈来源、用户类型、参与方式。
- 真实使用行为、成本支付行为、留存/复用信号。
- 购买决策者、实际使用者和受影响者之间的差异。

## 操作步骤

1. 区分反馈来源：共创用户、真实使用者、购买决策者、受影响者。
2. 标记反馈性质：口头认可、试用、主动复用、付费、推荐、替换旧方案。
3. 判断对方付出了什么成本：时间、金钱、数据、流程改变、组织协调。
4. 检查共创关系是否带来礼貌性反馈、样本偏差或过度配合。
5. 找出还缺的真实大盘证据。
6. 设计下一步验证：非共创用户试用、冷启动、付费、留存、替换行为。

## 关键自问

- 这个反馈来自真实用户，还是愿意帮我们的共创伙伴？
- 对方有没有付出真实成本？
- 他们是在评价想法，还是在替换旧方案？
- 反馈者和最终使用者是不是同一批人？
- 如果没有关系加成，这个方案还会被使用吗？

## 输出格式

```markdown
- 反馈来源：
- 用户类型区分：
  - 共创用户：
  - 真实使用者：
  - 购买决策者：
  - 受影响者：
- 当前证据：
- 成本支付行为：
- 偏差判断：
- 下一步验证：
```

## 常见误用

- 把共创用户的高热情当成市场需求。
- 把专家认可当成用户使用证据。
- 只看访谈好评，不看付费、复用和替代行为。
- 忽略购买者和使用者的目标差异。

## 失效信号

已有真实大盘行为数据；当前问题不是需求验证，而是明确需求后的交付质量；样本已经覆盖目标用户主群体。

## 常组合

- `事实-推断-假设分离`：拆反馈中的事实和推断。
- `用户朋友`：回到具体用户成本。
- `爆品聚焦`：从真实高强度场景中聚焦。
- `需求四问`：补齐需求判断。
