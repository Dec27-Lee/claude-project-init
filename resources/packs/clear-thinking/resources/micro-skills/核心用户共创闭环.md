# 核心用户共创闭环

> 动作类型：执行验证
> 场景标签：产品、增长、社区、内测、创业、工作

## 适用处境

产品仍在早期打磨或关键体验迭代中，团队缺少大规模市场反馈，但能接触到一批高动机、愿意投入时间、能提供具体反馈的核心用户；团队有能力持续响应，而不是只想借用户热情做宣传。

## 不适用处境

试用风险高到不应让用户承担；团队无法兑现迭代承诺；反馈涉及隐私、合规或安全，不能公开共创；目标用户是大众低参与用户，但样本只来自发烧友且无法外推；当前问题不是验证，而是明确需求后的交付质量。

## 输入

- 目标核心用户、招募标准和参与风险。
- 当前版本、反馈入口、迭代节奏和处理负责人。
- 反馈分层规则、公开回应机制和未采纳说明。
- 验证指标：留存、复用、推荐、二次反馈质量、问题关闭率。

## 操作步骤

1. 先定义核心用户：为什么这批用户更接近当前切口，而不是只因为他们热情或关系近。
2. 建立低摩擦反馈入口：问题、建议、投票、使用日志、访谈或社群都要能回到同一处理池。
3. 分层处理反馈：必须修复、体验优化、战略建议、个体偏好、噪声分别标记。
4. 公开优先级和节奏：说明本轮采纳什么、不采纳什么、为什么不采纳、何时复查。
5. 稳定迭代并回流结果：让用户能看到反馈被处理，形成信任而不是一次性调研。
6. 检查外推边界：定期用非共创用户、沉默用户或大众样本验证核心用户反馈是否偏移。

## 关键自问

- 这批核心用户代表的是当前切口，还是只代表高热情小圈子？
- 用户付出了什么成本？我们给了什么反馈和保障？
- 哪些反馈应进入产品优先级，哪些只是礼貌、情绪或个人偏好？
- 未采纳的意见有没有解释，还是让用户感到被利用？
- 如果没有关系和情怀加成，这个产品还会被继续使用吗？

## 输出格式

```markdown
## 共创对象
- 核心用户池：
- 招募理由：
- 用户风险/成本：

## 闭环机制
- 反馈入口：
- 优先级规则：
- 本轮采纳：
- 未采纳说明：
- 更新节奏：

## 验证
- 信任指标：
- 外推验证：
- 失败/切换信号：
```

## 常见误用

- 用用户投票外包战略和产品责任。
- 把粉丝热闹、社群活跃或专家认可当成真实质量。
- 用共创名义索取免费劳动，却不给回应、署名、保障或退出权。
- 只听高声量用户，忽略沉默用户、大众用户和实际付费者。
- 共创样本不外推，仍然直接推广到大盘。

## 失效信号

反馈长期无回应；团队无法维持承诺节奏；高声量用户绑架路线；核心用户与大众用户需求明显分裂；试用风险、隐私、合规或安全边界无法被保护；留存、复用和真实推荐没有改善。命中时切换到 `共创偏差`、`用户朋友`、`约束与风险扫描` 或停止共创。

## 最高风险边界

不能把用户当免费测试资源或营销素材；不能让用户承担未经说明的安全、隐私、资金、声誉和业务风险；不能用“共创”压制团队应承担的产品判断责任。

## 常组合

- `最小杠杆切口`：先明确要验证的核心切口。
- `共创偏差`：定期检查样本是否能外推。
- `用户朋友`：保护用户真实成本和信任关系。
- `口碑交叉验证`：把共创反馈与大盘口碑互证。
- `口碑飞轮`：共创闭环稳定后再设计增长机制。
