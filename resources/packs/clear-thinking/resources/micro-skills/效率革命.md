# 效率革命

> 动作类型：结构治理
> 场景标签：经营、增长、创业、长期判断、工作、生活

## 适用处境

流程很忙、工具很多、会议很多，但结果慢、返工多、交付不稳定，需要找最大损耗点并设计小闭环实验。

## 输入

- 当前输入、处理、输出、验收、反馈链路。
- 等待、返工、重复录入、信息断点、权限阻塞。
- 现有工具、角色分工和验收指标。

## 操作步骤

1. 画出从输入到验收的完整链路。
2. 标记等待、返工、重复劳动、信息断点、权限卡点。
3. 找出最大损耗点，而不是平均优化所有环节。
4. 判断损耗根因：规则不清、责任不清、工具割裂、信息缺口、验收滞后。
5. 设计一个小闭环实验：范围小、周期短、指标明确。
6. 设定验证方式：时间减少、返工减少、一次通过率提升或等待减少。

## 关键自问

- 当前最慢的是哪个环节，不是哪个人？
- 返工来自执行差，还是规则和验收不清？
- 工具增加是在减少工作，还是制造同步成本？
- 哪个小实验能最快验证效率瓶颈？
- 效率提升是否会转嫁成本给另一方？

## 输出格式

```markdown
- 当前链路：
- 最大损耗点：
- 损耗类型：等待 / 返工 / 重复 / 断点 / 权限
- 根因：
- 小闭环实验：
- 验收指标：
- 风险：
```

## 常见误用

- 直接上新工具，而不改规则和责任。
- 把人的努力不足误判为流程低效。
- 同时优化太多环节，无法验证效果。
- 为效率牺牲必要审查、沟通和安全边界。

## 失效信号

问题不是流程效率，而是方向错误、资源不足、目标冲突或外部约束；当前链路尚未稳定，不适合过早优化。

## 常组合

- `主要矛盾`：先确认瓶颈是不是效率。
- `敏捷债务`：识别快节奏下的返工债务。
- `责任状态`：处理验收和流转状态。
- `最小行动`：落成可验证实验。
