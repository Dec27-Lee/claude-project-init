# 组织叙事校准

> 动作类型：目标取舍
> 场景标签：AI、组织、产品、协作、工作

## 适用处境

战略、发布会、转型、AI 化战役、组织口号压到具体项目时，需要区分愿景、中期目标和本轮可交付事实。

## 输入

- 终局愿景、中局目标、本轮任务。
- 当前资源、证据、能力和时间窗口。
- 对外承诺、内部 KPI、项目真实状态。

## 操作步骤

1. 把叙事分成三层：终局愿景、中局目标、本轮任务。
2. 标出每层已有证据和缺口。
3. 判断本轮项目承接的是实验、试点、阶段能力，还是正式承诺。
4. 找出叙事越界点：把愿景当现状、把试点当规模化、把能力探索当结果承诺。
5. 设计降级表达：实验、试点、阶段目标、能力验证。
6. 明确本轮承诺和不承诺。

## 关键自问

- 这句话是在描述事实，还是在表达方向？
- 本轮项目能承担这个组织叙事吗？
- 哪些承诺会让团队被迫造假进展？
- 哪些内容可以说成实验，而不是正式能力？
- 对外/对内表达是否留下复盘和调整空间？

## 输出格式

```markdown
- 终局愿景：
- 中局目标：
- 本轮任务：
- 当前证据：
- 叙事越界点：
- 降级方式：实验 / 试点 / 阶段目标
- 本轮承诺：
- 本轮不承诺：
```

## 常见误用

- 用组织叙事否定真实进展需求。
- 把所有愿景表达都当成欺骗。
- 为了保守而不敢设定方向。
- 只改文案，不调整目标和验收。

## 失效信号

没有组织叙事压力；当前只是普通需求拆解；本轮已有明确事实、资源和验收，不存在越界表达。

## 常组合

- `事实-推断-假设分离`：区分事实与愿景。
- `敏捷债务`：识别叙事压力下的假进展。
- `完成检查`：把承诺落到验收。
- `质量裁决`：判断是否达到可发布质量。
