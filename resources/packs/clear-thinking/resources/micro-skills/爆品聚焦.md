# 爆品聚焦

> 动作类型：目标取舍
> 场景标签：经营、增长、创业、长期判断、工作、生活

## 适用处境

需求很多、资源有限、产品越来越散，需要选择 1-2 个主攻场景，形成强体验和可传播记忆点。

## 输入

- 候选需求、使用频次、痛感强度、传播性、成本。
- 用户类型、当前替代方案、最低交付边界。
- 资源约束和暂不服务对象。

## 操作步骤

1. 列出候选场景，而不是泛泛列功能。
2. 对每个场景判断痛感强度、频次、付费/复用/传播可能和交付成本。
3. 选择 1-2 个主攻场景，明确尖叫点。
4. 定义最低交付边界：做到什么才算有价值。
5. 明确砍掉项和暂不服务对象，防止继续发散。
6. 设定验证指标：复用、转化、留存、主动推荐或替代旧方案。

## 关键自问

- 哪个场景用户最愿意主动回来？
- 哪个点足够强，可以被一句话推荐？
- 哪些功能只是看起来完整，实际不增强尖叫点？
- 我们暂时不服务谁，才能把主场景做透？
- 聚焦后如何验证不是自嗨？

## 输出格式

```markdown
- 候选场景：
- 主攻场景：
- 尖叫点：
- 最低交付边界：
- 砍掉项：
- 暂不服务对象：
- 验证指标：
```

## 常见误用

- 把爆品理解成营销包装，而不是强场景体验。
- 只选高传播，不看交付质量和复用。
- 为了聚焦砍掉合规、基础体验或必要安全。
- 把个人偏好当成用户尖叫点。

## 失效信号

当前任务要求覆盖完整合规范围，不能聚焦砍项；产品仍在基础可用性阶段；核心用户和场景尚未定位。

## 常组合

- `用户朋友`：回到具体用户价值。
- `共创偏差`：验证尖叫点是否来自真实用户。
- `口碑飞轮`：把强体验转成推荐机制。
- `方案取舍`：处理多方案选择。
