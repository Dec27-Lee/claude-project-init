# 角色转换价值提案

> 动作类型：结构治理
> 场景标签：职业选择、组织、工作、成长

## 适用处境

个人想转岗、换职责、争取新角色或让组织新增岗位；组织没有现成入口，成败取决于少数关键决策人。需要把个人诉求翻译成组织问题、组织收益、授权边界和失败后手。

## 不适用处境

只是常规求职简历、已有明确晋升制度、劳动权益争议、薪资谈判或组织政治风险过高；用户没有能力或证据证明新角色能创造价值；当前更需要离职止损而非内部转化。

## 输入

- 当前岗位职责、痛点、瓶颈和用户想离开的原因。
- 组织当前缺口、已有相关方、关键决策人、可用资源。
- 用户能证明的能力、成果、样本和外部后手。

## 操作步骤

1. 区分个人不适与组织缺口：个人痛苦只能作为背景，不能当提案主体。
2. 写出新角色要解决的组织问题：减少什么损耗、补什么能力、连接哪些断点、创造什么可见价值。
3. 找关键决策路径：谁能批准、谁会受益、谁会受损、谁需要被提前沟通。
4. 设计授权边界：新角色的职责、接口、交付、考核、试运行周期和不负责事项。
5. 准备后手：如果不批准，内部其他路径、外部机会、降级方案和止损时间。
6. 把提案变成可验证试点，而不是一次性身份请求。

## 关键自问

- 我是在要求组织照顾我的痛苦，还是在解决组织确实存在的问题？
- 谁是唯一或关键决策人？我有没有找错人？
- 新角色上线后，哪些旧问题不会自动消失？
- 这个角色的边界如果不写清，会不会继续被塞回旧职责？
- 如果失败，我有没有不把自己困住的后手？

## 输出格式

```markdown
## 角色转换价值提案
- 当前岗位瓶颈：
- 组织真实缺口：
- 新角色价值：
- 关键决策人：
- 受益相关方：
- 受损/阻力相关方：
- 授权边界：
- 试运行周期与交付：
- 不负责事项：
- 失败后手：
- 停止/切换信号：
```

## 常见误用

- 只讲“我不想做现在的事”，不讲组织价值。
- 把模糊职位名当成转型，职责和考核仍然不清。
- 找错决策人或忽略受影响相关方。
- 没有后手，把谈判变成乞求。
- 用新角色逃避已有承诺、交接和能力短板。

## 失效信号

组织缺口不存在或证据不足；关键决策人不可达；角色边界无法被授权；试点风险不可承受；用户真正需要的是离开、求职、维权、休息或专业职业咨询。命中时转 `方案取舍`、`相关方损益与承接力地图`、`约束与风险扫描` 或 `切断力`。

## 最高风险边界

不能用本技能规避劳动权益、合规义务或必要交接；不能鼓励用户在无后手、无授权、无证据时进行高风险组织博弈。

## 常组合

- `相关方损益与承接力地图`：识别关键决策人、受益方和阻力方。
- `夹缝资源反推定位`：从现有弱资源反推可占位置。
- `会议签字固化承诺`：角色试点需要明确职责和承诺时。
- `方案取舍`：比较内部转岗、外部跳槽和暂时等待。
- `最小行动`：把提案落成一次试运行或沟通。
