# 问题分型

> 动作类型：问题诊断
> 场景标签：工作、生活、组织、个人、复盘

## 适用处境

不确定该直接执行、查证、调研、系统分析、止血、实验还是复盘；如果一开始选错处理模式，后续建议会变浅或变重。

## 输入

- 目标、影响范围、不确定性、可逆性、风险。
- 当前是否有损失扩大、外部承诺、长期后果或多方参与。
- 用户希望得到的产物和时间约束。

## 操作步骤

1. 收集五个判断维度：目标是否清楚、影响范围多大、不确定性多高、行动是否可逆、风险是否正在扩大。
2. 判断问题类型：简单执行、复杂专业问题、复杂系统问题、混沌止血问题或混合问题。
3. 给出本轮处理策略：直接做、先查证、先拆目标、先止血、先试点、先复盘。
4. 明确不适用方法：不能用一次性清单解决系统冲突，不能用长期规划处理正在扩大的事故。
5. 一旦分型能改变下一步行动，就停止继续分类。

## 关键自问

- 这是“不会做”，还是“不知道该不该做”？
- 现在最大风险是做错、做慢、做多，还是继续不动？
- 这个问题可逆吗，试错成本多高？
- 影响范围是个人、团队、组织、用户，还是长期资产？
- 如果只允许做一步，应该执行、查证、止血还是取舍？

## 输出格式

```markdown
- 问题类型：
- 判断依据：
- 本轮处理策略：
- 不适用方法：
- 下一步：
- 停止条件：
```

## 常见误用

- 把所有不清楚的问题都归为复杂系统问题，导致过度分析。
- 把所有工作任务都归为简单执行，忽略目标冲突。
- 分型后没有改变行动，只是多贴一个标签。
- 按领域分型，而不是按问题状态分型。

## 失效信号

类型判断不影响下一步；用户已经明确下一步，只需要执行；事实太少，连风险和可逆性都无法判断；分型越分越细但没有行动策略。

## 常组合

- `事实-推断-假设分离`：先校准事实再分型。
- `目标澄清`：目标不清时先澄清。
- `主要矛盾`：复杂问题分型后找主线。
- `最小行动`：分型后落到下一步。
- `质量裁决`：交付前判断是执行质量问题还是目标问题。
