# 解释关闭

> 动作类型：执行验证
> 场景标签：关系、个人、组织、生活、工作

## 适用处境

持续分析、反复解释、反复复盘，但没有进入行动、交接、止损或修复；需要判断是否停止分析并转入下一步。

## 输入

- 已解释清楚的事实、仍缺事实、行动延迟原因。
- 本轮责任边界、对方责任、不可控因素。
- 继续解释的收益和成本。

## 操作步骤

1. 列出已经解释清楚的部分和仍缺事实的部分。
2. 判断继续解释是否会显著提高判断质量。
3. 区分本轮责任、他人责任、系统约束和不可控因素。
4. 判断当前应进入行动、补关键事实、交接、止损还是沟通修复。
5. 设定最小行动和复盘点。
6. 对高风险关系、合规、承诺、风险披露，先完成必要告知和修复，再关闭解释。

## 关键自问

- 我还在补事实，还是在用解释拖延行动？
- 继续想下去会改变下一步吗？
- 哪些责任不是本轮应该承担？
- 哪些必要告知、道歉、修复或交接还没做？
- 最小行动是什么，什么时候复盘？

## 输出格式

```markdown
- 已解释清楚：
- 仍缺事实：
- 继续解释的收益：
- 继续解释的成本：
- 本轮责任：
- 非本轮责任：
- 关闭判断：停止分析 / 进入行动 / 补关键事实
- 最小行动：
- 复盘点：
```

## 常见误用

- 把解释关闭当成逃避责任或情绪切断。
- 在事实不足时强行停止判断。
- 对关系和承诺问题只做自我解释，不做沟通和修复。
- 把复杂问题一刀切归为“不是我的责任”。

## 失效信号

事实仍不足，关闭会导致误判；还存在必要风险告知、道歉、修复或交接；当前需要专业支持而不是自我解释。

## 最高风险边界

不能用解释关闭逃避必要告知、道歉、修复、交接或风险披露；不能把复杂关系中的责任一刀切；只有在事实足够、行动方向明确、继续解释不再增加判断质量时，才适合关闭。

## 常组合

- `事实-推断-假设分离`：确认事实是否足够。
- `切断力`：在需要退出时设计边界动作。
- `最小行动`：把解释转为行动。
- `完成检查`：判断是否能收口。
