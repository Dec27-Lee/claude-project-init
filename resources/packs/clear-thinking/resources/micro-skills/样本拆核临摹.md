# 样本拆核临摹

> 动作类型：产品表达
> 场景标签：产品、研发、设计、选型、工作、竞品、学习、替代方案

## 适用处境

理想样本太贵、不可买、不可直接复制，但希望在合规边界内获得相近核心效果；需要拆出机制而不是照抄表层。

## 不适用处境

核心价值来自资质、专利、版权、隐私数据、长期训练、独占关系或合规牌照；用户意图是复制外观、混淆来源或规避监管。

## 输入

- 理想样本、预算/资源、核心体验。
- 外观、结构、材料、流程、场景和反馈。
- 可替代组件、不可复制部分、知识产权和合规边界。

## 操作步骤

1. 拆样本：外观、结构、材料、流程、反馈、服务、场景，不把整体感受当成一个黑箱。
2. 标出真正产生效果的核心机制，以及只是品牌、价格、装饰或完美度的部分。
3. 标出不可复制部分：资质、专利、版权、隐私数据、独占关系、合规牌照。
4. 用可得资源替代核心机制，而不是照抄表层配置。
5. 做小样验证接近程度：哪些体验已足够，哪些断裂不可接受。
6. 形成升级、放弃或重新设计条件。

## 关键自问

- 这个样本真正产生效果的机制是什么？
- 哪些部分只是价格、品牌或外观带来的错觉？
- 哪些部分不能碰、不能仿、不能暗示？
- 低配替代是否会破坏核心体验？
- 小样验证证明的是核心机制，还是只证明表层相似？

## 输出格式

```markdown
- 理想样本：
- 核心机制：
- 可牺牲部分：
- 不可复制部分：
- 替代组件：
- 小样验证：
- 升级/放弃条件：
```

## 常见误用

- 抄外观而不拆机制。
- 低配模仿导致体验断裂。
- 忽略知识产权、隐私、资质或合规边界。
- 把不可复制资源伪装成“可替代设计”。

## 失效信号

核心价值来自资质、专利、版权、长期训练、独占关系、隐私数据或合规牌照时，不强行临摹；只要临摹会造成混淆、误认或侵权风险，应停止。

## 最高风险边界

不能把“拆核”变成规避合规、侵权复制、误导用户或混淆来源的包装。

## 常组合

- `体验锚点校准`：先确定真正打动人的体验指标。
- `方案取舍`：比较自研、购买、替代、放弃。
- `质量裁决`：判断小样是否达标。
- `约束与风险扫描`：涉及合规或知识产权时。
