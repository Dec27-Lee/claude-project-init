# 约束与风险扫描

> 动作类型：风险边界
> 场景标签：工作、生活、组织、个人、复盘

## 适用处境

涉及文件修改、删除、迁移、发布、部署、对外承诺、权限变更、长期资料入口、不可逆选择或高成本行动；行动影响多人、客户、用户、财务、合规、信誉或长期资产。

## 不适用处境

低风险、可逆、局部、小范围任务，扫描不会改变行动；用户只要概念解释；已经有更专业的安全、法律、财务、代码审查流程；时间紧急且扫描会妨碍安全处置。

## 输入

- 操作对象、影响范围、可逆性、验证方式。
- 用户明确指令、项目规则、权限边界。
- 隐私、安全、合同、财务、法律、生产系统、外部发布等风险项。

## 操作步骤

1. 确认操作对象：文件、系统、关系、承诺、资金、权限、发布物或长期资产。
2. 判断影响范围：只影响自己、团队内部、用户/客户、外部公众或长期制度。
3. 判断可逆性：可立即撤回、可恢复但有成本、很难恢复、不可逆。
4. 区分硬约束与软偏好：权限、法律、合同、安全、隐私、数据、明确指令是硬约束；风格、节奏、习惯多为软偏好。
5. 分级风险：低风险可直接执行，中风险需防护，高风险需确认、备份、测试、审阅或升级。
6. 设计防护动作：确认、备份、测试、只读检查、灰度、回滚、延迟发布、记录决策。
7. 给出执行结论：直接执行、先确认、先防护、暂停升级。

## 关键自问

- 这件事改动的对象是什么，谁会受影响？
- 如果做错，能不能恢复，多久恢复，谁承担代价？
- 有没有明确的用户指令、项目规则或权限边界？
- 是否涉及隐私、安全、合同、财务、法律、信誉或生产系统？
- 是否会改变长期资料入口、规范、配置或公共行为？
- 有没有一个更小、更可逆的动作可以先做？
- 做完如何知道没有破坏东西？

## 输出格式

```markdown
## 约束与风险扫描
- 操作对象：
- 影响范围：个人 / 团队 / 用户 / 外部 / 长期资产
- 可逆性：高 / 中 / 低 / 不可逆
- 硬约束：
- 软偏好：
- 风险等级：低 / 中 / 高
- 防护动作：确认 / 备份 / 测试 / 审阅 / 回滚 / 记录
- 执行结论：可直接执行 / 需先确认 / 需先防护 / 暂停升级
- 失效或停止信号：
```

## 常见误用

- 对低风险小事过度扫描，拖慢执行。
- 把软偏好误当硬约束。
- 只列风险，不给防护动作和执行结论。
- 忽视可逆性，把所有风险等同处理。
- 用户明确禁止修改时仍讨论如何修改。
- 用“风险可控”替代实际测试、备份或确认。

## 失效信号

低风险小任务，扫描不改变行动；扫描后仍没有执行结论；风险项无法对应防护动作；用户需要的是价值判断而非执行风险判断；已进入专业审查流程。

## 最高风险边界

不越过用户明确的只读、禁止修改、禁止发布、禁止外传指令；涉及隐私、安全、法律、财务、生产系统、外部发布时默认高风险；不在缺少权限和恢复手段时做不可逆操作。

## 常组合

- `切断力`：退出、停项、拒绝前做后果扫描。
- `完成检查`：执行后验证是否达成且没有破坏边界。
- `事实-推断-假设分离`：区分真实约束和脑补风险。
- `最小行动`：把高风险行动降为小步可逆实验。
- `承接责任检查`：明确风险由谁承担。
- `质量裁决`：交付前判断质量风险是否可接受。
