# 责任位置推进判断

> 动作类型：结构治理
> 场景标签：工作、协作、项目、产品、执行

## 适用处境

经手事项卡住、群里问题无人承接、跨角色任务没人说了算，或自己不确定应该主责推进、协作补位，还是只提醒风险并退出。

## 不适用处境

- 已有明确流程、负责人和升级路径，且不允许越级协调。
- 涉及合规、安全、权限审批、法律责任或人事处分，需要按正式流程处理。
- 事实未清，只是想抢答、抢功或替别人做承诺。
- 当前问题不是推进卡住，而是目标不清、需求不清或方案未定。

## 输入

- 事项名称、当前状态、卡住点。
- 名义负责人、实际推动者、受影响方、关键资源方。
- 自己与事项的关系：结果责任、局部责任、信息提供、资源协调、旁观者。
- 可用权限、不可越过的流程和最晚反馈时间。

## 操作步骤

1. 判定结果责任：这件事最终由谁对结果负责？如果没人能说清，先把“负责人不清”作为当前阻塞。
2. 判定自己位置：我是结果负责人、局部责任人、资源连接者、事实提供者，还是只是被抄送/旁观？
3. 匹配推进方式：
   - 结果负责人：直接给下一步、时间点和验收口径。
   - 局部责任人：完成自己部分，并把依赖和风险同步给主责人。
   - 资源连接者：只连接资源和事实，不替主责人承诺结果。
   - 旁观者：只在高风险遗漏时提醒一次，不持续介入。
4. 写出补位边界：我能补什么、不能补什么、需要谁确认。
5. 设失败信号：到什么时间点仍无人响应、风险扩大或责任继续不清时，升级/停手/转正式流程。

## 关键自问

- 我是在承担自己该承担的结果，还是在替别人做承诺？
- 这件事卡住是因为没人负责、资源缺口、事实缺口，还是决策缺口？
- 我补位后，主责人会更清楚责任，还是责任更混乱？
- 如果我不推进，最先受影响的是谁？影响是否值得我提醒或升级？
- 有没有权限、合规或安全边界要求我停止介入？

## 输出格式

```markdown
## 责任位置
- 事项：
- 结果负责人：
- 我的责任位置：主责 / 局部责任 / 资源连接 / 事实提供 / 旁观提醒

## 推进方式
- 我下一步做：
- 需要主责人确认：
- 需要连接的资源：
- 不由我承诺的内容：

## 失败信号
- 到____仍____时，升级/停手/转流程：
```

## 常见误用

- 把主动推进变成越权指挥、抢功或替别人承诺结果。
- 用“我不是负责人”合理化完全不提醒、不补位。
- 只催别人，不说明自己能提供什么事实、资源或局部交付。
- 在责任和权限不清时直接给外部承诺。

## 失效信号

开始讨论谁对谁错而不是下一步承接；责任位置无法通过事实确认；事项进入正式审批、合规、安全或法律流程；自己没有信息也没有权限，只是在增加沟通噪声。命中时切换到 `约束与风险扫描`、`会议签字固化承诺` 或正式流程。

## 最高风险边界

不能用本技能绕开组织授权、合规审批或安全流程；不能替他人承担法律、资金、合同、人事和对外承诺责任；不能用“主动”要求弱权限角色背锅。

## 常组合

- `相关方损益与承接力地图`：相关方多、承接条件和损益不清时。
- `会议签字固化承诺`：口头承诺需要固化为责任、时间和退出条件时。
- `完成检查`：推进后检查事项是否真的闭环。
- `信息缺口定位`：卡住原因是缺事实或缺材料时。
