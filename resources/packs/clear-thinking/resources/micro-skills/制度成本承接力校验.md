# 制度成本承接力校验

> 动作类型：结构治理
> 场景标签：组织、治理、产品、平台、公共议题、工作

## 适用处境

制度、政策、流程、平台规则或组织机制的目标看起来正确：长期安全、合规、公平、效率、保障、环保、质量；但执行成本可能转嫁给承接力最弱的人、团队、商户、用户或基层组织。需要同时检查目标正当性和成本承接力。

## 不适用处境

规则属于不可让渡的安全、法律、合规、伦理底线；缺少事实数据，只能表达立场；当前任务只是执行已确定规则；问题需要专业法律、财务、公共政策或安全评估。

## 输入

- 制度目标、执行对象、执行动作和强制程度。
- 显性成本、隐性成本、成本承担者和收益者。
- 各方现金、时间、能力、组织程度、替代方案和退出权。
- 安全、合规、长期资金池、公共风险等不可忽略的约束。

## 操作步骤

1. 先承认制度目标：它试图解决什么风险、保障、效率或长期问题。
2. 拆显性成本和隐性成本：钱、时间、流程、证明、等待、机会、尊严、退出和信任。
3. 画成本转嫁链：名义承担者是谁，实际可能通过涨价、降薪、外包、绕行、压缩服务等方式转给谁。
4. 找最弱承接方：谁最缺现金、时间、话语权、替代方案或组织能力。
5. 判断挤出信号：谁会因为规则被迫退出、转入灰色路径、降低生活/服务质量或承担不可承受风险。
6. 设计缓冲：差异化费率、过渡期、财政/组织补贴、分层执行、豁免条件、试点回流和申诉机制。
7. 若目标不能让渡，输出“必须执行 + 如何补承接条件”，而不是简单反对。

## 关键自问

- 这个制度的目标正当性和执行承接力是否被混在一起了？
- 谁名义上承担成本，谁实际上会被转嫁成本？
- 哪个群体不是不愿意，而是没有能力承接？
- 有没有把未来确定性建立在当下生存不确定性之上？
- 如果必须执行，最小缓冲和申诉机制是什么？

## 输出格式

```markdown
## 制度成本承接力校验
- 制度/规则目标：
- 不可让渡约束：
- 显性成本：
- 隐性成本：
- 成本转嫁链：
- 最弱承接方：
- 挤出/驱逐信号：
- 缓冲方案：
- 试点/回流指标：
- 失败/切换信号：
```

## 常见误用

- 只因承接困难就否定所有公共规则、安全底线或长期保障。
- 只谈制度目标，忽视成本转嫁和弱承接方。
- 只凭情绪推断政策/规则动机，不做事实校准。
- 把弱承接方描述成“低端”“落后”并合理化驱逐。
- 用“精准、科学、高效”包装没有资源支持的强制执行。

## 失效信号

缺少成本和承接事实；规则本身是刚性安全/合规底线且不能延迟；问题需要专业政策、法律或财务评估；输出开始变成立场宣泄或人群标签。命中时切换到 `事实-推断-假设分离`、`约束与风险扫描`、`可监督承诺边界` 或专业支持。

## 最高风险边界

不能用承接力不足为违法、侵害、安全事故或逃避基本义务辩护；不能把公共政策判断简化成情绪立场；不能制造对弱势主体的标签化和驱逐合理化。

## 常组合

- `相关方损益与承接力地图`：需要列出多类相关方及承接条件时。
- `可监督承诺边界`：制度承诺需要指标、监督和复核时。
- `约束与风险扫描`：涉及法律、合规、安全和不可逆后果时。
- `样本试点回流循环`：需要先用试点验证成本和缓冲机制时。
- `用户朋友`：规则影响具体用户或基层角色的真实处境不清时。
