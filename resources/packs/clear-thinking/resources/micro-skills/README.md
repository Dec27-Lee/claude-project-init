# 微技能目录

本目录采用“一微技能一文件”。文件不再按工作/生活/产品/经营等使用场景分目录，避免把真实问题框死到单一场景。

运行时流程：先由 `ROUTER.md` 判断问题缺哪类判断动作，再到 `MICRO_SKILLS.md` 按动作类型、触发信号、产出、场景标签和常组合定位具体文件，只读取命中的少量微技能。

场景标签只用于检索和组合，不作为调用边界；同一个问题可以同时调用事实校准、目标取舍、结构治理、关系承接、执行验证等不同类型微技能。

## 微技能内容分层

微技能不追求统一长度，而按调用风险和频率分层增强：

1. **高频底座类**：目标澄清、事实校准、主要矛盾、方案取舍、最小行动、完成检查、信息缺口定位等，必须包含执行步骤、关键自问和输出格式。
2. **高风险边界类**：资金、合伙、危机、责任、关系、合规、可信度、临摹等，必须额外包含不适用处境、常见误用、最高风险边界和升级/停止条件。
3. **轻量表达类**：字段、页面、状态、组件、信息分档等，可以保持较短，但仍应有稳定输出格式和误用边界。

## 推荐文件结构

高频或高风险微技能建议使用：

```markdown
---
name: 微技能名
action_type: 动作类型
scene_tags: [工作, 生活, 产品]
output: 该技能产生的中间结论
---
# 微技能名

> 动作类型：动作类型
> 场景标签：工作、生活、产品

## 适用处境
## 不适用处境
## 输入
## 操作步骤
## 关键自问
## 输出格式
## 常见误用
## 失效信号
## 最高风险边界
## 常组合
```

低风险轻量微技能可以省略部分小节，但不能省略“适用处境、输入、操作、产出/输出格式、失效信号”。

## 维护原则

- 扩充微技能时只增加运行层判断动作，不写来源材料、人物、剧情、语录或权威背书。
- `MICRO_SKILLS.md` 只负责定位候选；采用微技能必须读取对应文件。
- 高风险技能宁可多写失效信号，也不要给出可被误用的泛化建议。
- 常组合应尽量与 `MICRO_SKILLS.md` 中的“常组合”保持一致；如新增强组合，应同步评估是否需要更新索引。
