# 长期复利

> 动作类型：目标取舍
> 场景标签：经营、增长、创业、长期判断、工作、生活

## 适用处境

短期收益和长期能力、信誉、健康、关系、资产沉淀发生冲突，需要判断做、不做、延后、降级或设边界做。

## 输入

- 短期收益、短期成本和当前约束。
- 长期资产：能力、信用、关系、系统、数据、品牌、健康。
- 不可承受损害和阶段复盘点。

## 操作步骤

1. 拆出短期收益和短期成本。
2. 判断这件事是否沉淀长期资产：能力、信用、关系、系统、数据、品牌、健康。
3. 区分一次性收益和可复用复利收益。
4. 标记不可承受损害：合规、安全、健康、核心关系、用户信任、关键承诺。
5. 判断节奏：关键条件是否已具备、优势/劣势是否会随时间放大、外部窗口是否短暂，选择速决、持久、暂缓或止损。
6. 结合当前生存约束，判断做、不做、延后、降级或设边界做。
7. 设置复盘点和停止信号，避免长期口号无限占用资源。

## 关键自问

- 做完以后留下什么可复用资产？
- 这只是一次性收益，还是会让下一次更容易？
- 为了长期收益，会不会牺牲当前必须履行的责任？
- 当前是否有短期生存约束压倒长期投入？
- 什么时候复盘，什么信号说明该停止？
- 当前更适合速决、持久、暂缓还是止损，依据是条件、窗口还是承受力？

## 输出格式

```markdown
- 短期收益：
- 长期资产：
- 一次性消耗：
- 不可承受损害：
- 当前约束：
- 建议：做 / 不做 / 延后 / 降级 / 设边界做
- 复盘点：
- 停止信号：
```

## 常见误用

- 用长期复利为拖延、完美主义或不交付辩护。
- 忽略短期现金流、责任和承诺。
- 把任何学习、沉淀都包装成长期资产。
- 没有复盘点，长期投入变成无底洞。

## 失效信号

短期生存约束压倒长期投入；任务有明确截止和刚性责任；长期资产无法被定义或验证。

## 最高风险边界

不能用长期复利为拖延、完美主义、资源占用或不交付辩护；不能为了长期资产牺牲当前必须履行的责任、安全、合规和承诺；长期判断必须有阶段复盘点和停止信号。

## 常组合

- `阶段轮廓与转折点管理`：长期投入需要阶段地图、转入条件和止损线时。
- `方案取舍`：处理不同路径的代价。
- `敏捷债务`：识别短期迭代是否积累债务。
- `口碑飞轮`：把信任和体验作为长期资产。
- `完成检查`：设置阶段验收。
