# 口碑飞轮

> 动作类型：增长信任
> 场景标签：经营、增长、创业、长期判断、工作、生活

## 适用处境

增长主要靠宣传、投放或活动，用户复用和主动推荐不足，需要找到真实口碑的来源和破坏点。

## 输入

- 核心体验、复用场景、失败体验。
- 留存、复用、推荐、投诉、退款或沉默流失信号。
- 用户愿意告诉别人或不愿意推荐的理由。

## 操作步骤

1. 找到用户愿意再次使用的核心体验。
2. 找到用户愿意主动推荐的理由：省时、省钱、避险、身份、情绪、结果确定性。
3. 识别破坏口碑的点：失败体验、承诺落差、隐私风险、售后差、学习成本高。
4. 优先修复最伤信任的点，而不是先设计传播激励。
5. 判断口碑杠杆：可复用场景、可讲述故事、可展示结果、低风险分享路径。
6. 设定验证信号：自然复用、主动推荐、非激励分享、正向评价质量。

## 关键自问

- 用户为什么会主动回来？
- 用户愿意向朋友推荐哪一句话？
- 哪个失败体验会让用户再也不推荐？
- 推荐是因为真实价值，还是因为奖励和压力？
- 我们是否在消耗用户关系换增长？

## 输出格式

```markdown
- 核心体验：
- 复用理由：
- 推荐理由：
- 破坏口碑点：
- 优先修复项：
- 口碑杠杆：
- 验证信号：
```

## 常见误用

- 把裂变、返利、邀请奖励当成口碑。
- 在核心价值未成立前设计传播机制。
- 只看推荐数量，不看推荐后的满意度。
- 为传播牺牲隐私、关系和用户尊严。

## 失效信号

还没到增长阶段，核心价值未成立；用户留存极差，先修产品；增长目标来自短期曝光而非长期信任。

## 最高风险边界

不能用激励、社交压力或暗黑模式制造伪推荐；不能牺牲隐私、联系人关系、用户尊严换传播；口碑机制必须建立在真实价值、可复用体验和用户自愿表达上。

## 常组合

- `用户朋友`：保护推荐中的长期信任。
- `爆品聚焦`：强化可推荐尖叫点。
- `长期复利`：避免短期增长透支口碑资产。
- `默认值制度`：审计默认分享和默认邀请。
