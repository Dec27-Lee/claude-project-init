# 资本控制权校准

> 动作类型：结构治理
> 场景标签：经营、增长、创业、长期判断、工作、生活、合伙、家庭资产、项目治理

## 适用处境

多人、多钱、多责任项目启动前，信任、出资、控制、风险、债务、名义安排和退出条件混在一起。

## 不适用处境

意图是规避监管、逃避债务、代持背锅、利益输送或让不知情一方承担风险；已进入法律、税务、财务实操阶段，需要专业支持。

## 输入

- 出资方、控制方、执行方、顾问方、受益方。
- 股权/债权/担保/名义安排、亏损承担和债务责任。
- 决策权、退出条件、法律手续、知情同意情况。

## 操作步骤

1. 列出谁出钱、谁控股、谁实际执行、谁只是顾问、谁获得收益。
2. 标出亏损、债务、担保、名义责任由谁承担。
3. 检查风险承担与控制权是否匹配，避免只承担风险不掌握方向。
4. 写清重大事项决策、分红、回购、债权债务、退出条件和争议处理。
5. 把名义安排、担保和退出条件写成真实法律/财务手续。
6. 标出不可接受结构，并在高风险处停止推进。

## 关键自问

- 谁出钱，谁说了算，谁背风险？
- 是否有人承担风险却没有控制权？
- 名义安排和真实安排是否一致？
- 退出、回购、债务、担保是否写清？
- 参与方是否真实知情、自愿并能拒绝？

## 输出格式

```markdown
- 出资结构：
- 控制权结构：
- 执行责任：
- 风险承担：
- 法律/财务手续：
- 退出条件：
- 不可接受安排：
- 专业支持需求：
```

## 常见误用

- 只谈信任，不谈控制权和风险承担。
- 用亲密关系替代法律、财务和退出手续。
- 让名义方承担实际不可控风险。
- 把“顾问/帮忙”做成事实控制或隐性担保。

## 失效信号

规避监管、逃避债务、利益输送、挂名背锅、弱势方无法知情同意时停止；只要存在挂名、担保、债务或代持安排，默认需要专业审查。

## 最高风险边界

不能设计或合理化让一方背风险、无控制、不了解真实后果或无法拒绝的结构。

## 常组合

- `会议签字固化承诺`：把关键安排进入确认记录。
- `约束与风险扫描`：补充风险和不可碰线。
- `承接链结构设计`：涉及长期扶持、生态合作或资源投入时。
- `朋友请求边界问询`：合伙请求来自熟人关系时。
