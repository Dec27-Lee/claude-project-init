# 会议签字固化承诺

> 动作类型：结构治理
> 场景标签：工作、生活、组织、个人、复盘、跨部门、合伙、项目启动

## 适用处境

多人协作、合伙、项目启动、跨部门承诺中，大家口头同意但责任、分工、时间、保密、退出、争议处理不清。

## 不适用处境

低风险小事，轻量同步即可；信息不充分，无法有效确认；参与方无法自由拒绝或存在明显权力压力；重大法律/财务后果需要专业文件。

## 输入

- 参与方、争议点、承诺事项、分工、时间表。
- 保密内容、退出条件、记录方式、权力关系。
- 哪些事项需要法律、财务或组织正式流程。

## 操作步骤

1. 会前列明议题、争议点、待确认事项和必要材料，让反对意见有入口。
2. 会中先互通事实、数量边界和各方限制，再讨论判断；允许异议、保留意见和未确认事项进入记录，不用沉默代表同意。
3. 区分中心任务和配合任务，写清出资、分工、硬指标、时间、保密、退出和争议处理。
4. 让每个人在知情、自愿、可拒绝的前提下确认。
5. 后续执行按记录处理，不按记忆和情绪重谈。
6. 若文本会产生重大法律或财务后果，切换到专业支持。

## 关键自问

- 谁承担结果，谁拥有决策权？
- 哪些口头承诺需要转成书面记录？
- 有没有人是在压力下确认？
- 哪些事项仍未确认，不能假装已同意？
- 纪要是否被误当成合同或法律手续？

## 输出格式

```markdown
- 会议目标：
- 已确认事项：
- 未确认事项：
- 责任人：
- 时间表：
- 保密/退出：
- 确认方式：
- 争议处理：
```

## 常见误用

- 用签字压制异议。
- 信息未充分披露就要求确认。
- 把纪要当合同，忽略必要法律手续。
- 用“会都开了”替代真实知情和自愿。

## 失效信号

信息不充分、权力不平等、被迫确认、弱势方无法拒绝或低风险小事不必过度治理时不用；确认文本会产生重大法律或财务后果时，应切换到专业支持。

## 最高风险边界

不能用书面确认包装不知情、不自愿、不可拒绝或信息不充分的承诺。

## 常组合

- `责任状态`：明确状态和责任人。
- `资本控制权校准`：涉及出资、控制权、亏损和退出时。
- `完成检查`：执行后检查承诺是否兑现。
- `约束与风险扫描`：承诺涉及风险或不可逆后果时。
