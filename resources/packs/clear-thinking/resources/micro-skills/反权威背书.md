# 反权威背书

> 动作类型：方法建设
> 场景标签：方法建设、学习、蒸馏、复盘

## 适用处境

输出容易变成“某书/某人/某公司/某案例说过”，来源替代了当前事实、目标和验证；需要把权威内容降级为观察角度或候选假设。

## 输入

- 材料观点、来源表达、当前问题。
- 当前事实、目标、约束、风险和验证方式。
- 是否需要资料综述或当前应用判断。

## 操作步骤

1. 找出输出中的权威成分：某人说、某书说、某公司做过、某案例证明。
2. 将权威内容降级：从依据降级为观察角度、候选假设或类比提醒。
3. 回到当前问题：当前事实是什么，目标是什么，约束是什么，风险是什么。
4. 重新生成判断依据：只用当前事实、假设和验证方式支撑结论。
5. 若必须提及来源，只说明它是参考角度，不作为背书。
6. 输出去来源化判断，并标出待验证项。

## 关键自问

- 如果删掉书名、人名、公司名，这个判断还成立吗？
- 当前事实是否足以支持结论？
- 我是在解释问题，还是展示自己读过材料？
- 来源里的做法是否依赖特殊资源、时代、身份或组织条件？
- 用户要的是资料综述，还是当前问题的应用判断？
- 有没有把案例相似误当成因果相同？

## 输出格式

```markdown
## 反背书检查
- 原背书表达：
- 降级后角色：观察角度 / 候选假设 / 类比提醒
- 当前事实依据：
- 当前假设：
- 验证方式：
- 去来源化判断：
```

## 常见误用

- 完全禁止来源，导致资料综述任务失真。
- 表面不提来源，但仍照搬来源结论。
- 把权威观点改写成自己的话，却没有当前事实支撑。
- 用“反权威”掩盖不查证。
- 把所有经验类知识都视为无效。

## 失效信号

用户明确只要资料综述，不要求应用；当前任务需要准确引用原文、出处、观点谱系；输出已经完全基于当前事实、没有权威依赖；去来源化后只剩空话，说明当前事实不足。

## 最高风险边界

反权威背书不能变成反专业、反证据；医学、法律、安全、财务、合规等高风险领域不能用“去权威化”绕开专业标准，应区分权威背书与可验证证据/合规要求。

## 常组合

- `判断动作提取`：材料蒸馏时先提动作再去背书。
- `微技能封装`：封装前检查是否依赖来源。
- `事实-推断-假设分离`：重建当前判断依据。
- `质量裁决`：检查输出是否靠名词和案例撑场。
- `完成检查`：最终反教条检查。
