# 体验锚点校准

> 动作类型：事实校准
> 场景标签：产品、研发、设计、选型、工作、消费、审美、技术选型

## 适用处境

进入陌生消费、产品选型、技术选型、审美判断或体验设计时，信息很多但缺少真实判断基准，需要先建立可观察的体验锚点。

## 不适用处境

没有真实体验条件，只能看二手材料；样本场景与自己的长期使用场景差异过大；当前主要问题是预算、合规、供应能力或组织取舍。

## 输入

- 高质量样本、自己的真实体验、市场/方案对照样本。
- 使用场景、预算/资源约束、长期维护条件。
- 当前要做的购买、选型、设计或改造判断。

## 操作步骤

1. 选择一个真实高质量样本，不以热度或价格自动等同质量。
2. 亲自体验，并记录具体差异：流程、反馈、稳定性、细节、服务、材料、速度、情绪负担。
3. 把感受翻译成可观察指标，而不是停留在“高级/舒服/顺滑”。
4. 用指标比较多个现实选项，区分必须项、加分项和个人偏好。
5. 检查锚点是否可复现、可承受、符合长期使用场景。
6. 更新指标后再决定购买、采用、临摹或放弃。

## 关键自问

- 打动我的到底是外观、流程、反馈、稳定性，还是服务承诺？
- 这个体验在我的实际使用场景中是否仍成立？
- 哪些指标是必须项，哪些只是偏好？
- 我能否用低成本方式验证长期体验？
- 这个样本是否因为价格、品牌或极端条件才成立？

## 输出格式

```markdown
- 锚点样本：
- 触发差异：
- 可观察指标：
- 对照样本：
- 必须项/偏好项：
- 预算/资源限制：
- 采用判断：
- 待验证点：
```

## 常见误用

- 把昂贵、稀缺或极端样本当成唯一标准。
- 沉迷审美偏好，忽略使用场景和长期维护。
- 用一次体验替代长期验证。
- 指标无法观察或复现，却被当成事实。

## 失效信号

第一样本来自极端场景、不可复现、预算完全不可承受，或长期使用场景不同，不把它当唯一标准；体验指标无法观察或复现时降级为假设。

## 最高风险边界

不能把不可复现、不可承受或场景错位的体验当作普遍判断基准。

## 常组合

- `方案取舍`：围绕指标比较不同方案。
- `样本拆核临摹`：需要从锚点中拆出核心机制时。
- `信息缺口定位`：缺少真实体验或对照证据时。
- `最小行动`：设计低成本试用或小样验证。
