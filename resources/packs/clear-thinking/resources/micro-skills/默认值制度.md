# 默认值制度

> 动作类型：结构治理
> 场景标签：AI、组织、产品、协作、工作

## 适用处境

AI 自动排序、提醒、推荐、摘要、权限、折叠、勾选、分配、通知等默认行为会改变用户行为、责任或资源分配。

## 输入

- 默认规则、用户角色、可解释性、可调整性、退出路径。
- 默认值影响对象、保护对象、潜在损害。
- 是否涉及权限、费用、通知、隐私、AI 判断或对外展示。

## 操作步骤

1. 列出所有默认行为：排序、推荐、提醒、摘要、权限、折叠、勾选、分配、通知。
2. 判断默认值保护谁、服务谁、影响谁。
3. 检查默认值是否可解释、可调整、可退出。
4. 识别默认值是否改变用户行为或组织责任。
5. 判断默认值是否偏向某一方利益或隐藏关键成本。
6. 给出审计结论：保留、弱化、显式确认、改成用户选择、禁用。

## 关键自问

- 默认值是在帮用户省力，还是替系统省解释？
- 用户不改默认值会承担什么后果？
- 默认值是否偏向某一方利益？
- 用户是否知道自己正在接受默认规则？
- 退出路径是否真实可用，而不是形式存在？

## 输出格式

```markdown
- 默认项：
- 默认规则：
- 影响对象：
- 保护对象：
- 潜在损害：
- 可解释性：
- 可调整性：
- 可退出性：
- 建议：保留 / 弱化 / 显式确认 / 改为选择 / 禁用
```

## 常见误用

- 把默认值当成小交互，不看制度后果。
- 用默认勾选、默认推荐、默认提醒引导用户做非自愿动作。
- 默认权限过大，事后让用户承担风险。
- 默认折叠关键信息，降低用户知情质量。

## 失效信号

没有默认规则；默认值不影响行为、责任、风险或资源分配；当前问题只是文案或视觉优先级。

## 最高风险边界

不能通过默认值诱导用户放弃权利、隐私、时间、金钱或解释机会；不能把高风险行为设置为静默默认；涉及权限、通知、AI 判断、对外展示、费用、承诺时，必须显式可知、可调、可退。

## 常组合

- `组件权力`：审计默认排序、默认 Feed、默认待办。
- `责任状态`：审计自动确认和自动状态。
- `用户朋友`：判断是否损害长期信任。
- `长期复利`：避免短期转化透支资产。
