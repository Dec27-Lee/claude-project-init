# 定位四问

> 动作类型：目标取舍
> 场景标签：AI、组织、产品、协作、工作

## 适用处境

方向很大，但不知道先服务谁、在哪用、替代什么、为什么是自己做；需要用定位排除需求，而不是写愿景口号。

## 输入

- 目标用户、使用场景、当前替代方案、团队优势。
- 不服务对象、不可承诺能力、阶段性资源。
- 需要被定位约束的需求或方案。

## 操作步骤

1. 回答为谁：目标用户/客户/团队/角色是谁，不服务谁。
2. 回答在哪用：高频场景、关键时刻、触发条件。
3. 回答替代什么：工具、流程、人力、习惯、认知成本或风险承担。
4. 回答为什么是我：独特能力、资源、渠道、信任、数据、速度或上下文。
5. 用定位排除需求：哪些现在做，哪些后做，哪些不做。
6. 形成一句可执行定位，并检查它是否能指导取舍。

## 关键自问

- 目标对象是否具体到能想象使用场景？
- 这个场景是刚需、高频、高痛，还是只是泛泛有用？
- 用户现在不用我时怎么解决？
- 我替代的是哪个环节？
- 为什么我方比别人更适合做？
- 这个定位能排除需求吗？

## 输出格式

```markdown
## 定位四问
- 为谁：
- 在哪用：
- 替代什么：
- 为什么是我：

## 不服务
- 不服务对象：
- 不进入场景：
- 不承诺能力：

## 定位句
- 为____在____场景下，替代____，因为____。

## 需求排除
- 现在做：
- 暂不做：
```

## 常见误用

- 把定位写成愿景口号。
- 只说目标用户，不说替代方案。
- 只说市场需要，不说为什么是我。
- 不明确不服务谁，导致范围膨胀。
- 定位不能指导需求取舍。

## 失效信号

已有清晰定位且本轮只做执行；四问答案过于抽象，无法排除需求；目标对象、场景、替代方案任一缺失；“为什么是我”只剩热情，没有优势证据。

## 常组合

- `顺势判断`：先判断方向，再确定切入点。
- `需求四问`：定位落到需求表达。
- `爆品聚焦`：从定位收敛到核心体验。
- `方案取舍`：根据定位选方案。
- `不可能三角识别`：定位取舍时说明牺牲项。
