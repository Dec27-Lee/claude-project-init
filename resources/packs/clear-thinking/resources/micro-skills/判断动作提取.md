# 判断动作提取

> 动作类型：方法建设
> 场景标签：方法建设、学习、蒸馏、复盘

## 适用处境

需要从书籍、PDF、个人素材、访谈、课程、案例或复盘中吸收思想，但目标不是摘要、金句或角色扮演，而是提取可迁移判断动作。

## 输入

- 材料摘要、关键场景、行动者如何判断和行动。
- 原始问题、约束、取舍、行动、反馈。
- 可能迁移到工作/生活/产品/经营中的触发处境。

## 操作步骤

1. 明确材料中的关键场景：行动者在什么处境下判断。
2. 不摘概念，寻找动作：如何观察问题、识别主次、设置边界、选择行动、验证修正。
3. 抽离来源语境：去掉人物、行业、故事、口号和特殊身份条件。
4. 转写为一般判断动作：输入是什么，处理动作是什么，产出是什么。
5. 检查迁移性：能否用于其他场景，使用边界是什么。
6. 产出候选微技能素材，而不是资料摘要。

## 关键自问

- 这里真正可复用的是观点，还是判断动作？
- 这个动作依赖特定身份、资源或时代背景吗？
- 如果去掉来源，动作还成立吗？
- 它解决的是事实、目标、取舍、执行、验证还是复盘问题？
- 它什么时候不该用？
- 能不能写成输入—操作—产出—失效信号？

## 输出格式

```markdown
## 判断动作提取
- 原始场景：
- 观察动作：
- 主次判断：
- 边界设置：
- 行动方式：
- 验证方式：

## 可迁移动作
- 动作名称：
- 适用处境：
- 输入：
- 操作：
- 产出：
- 失效信号：
```

## 常见误用

- 摘录金句或概念，而不是提取动作。
- 保留太多来源语境，导致无法迁移。
- 把价值判断包装成方法。
- 只提炼“应该怎样”，没有输入和失效信号。
- 从纯事实材料中强行提炼方法。

## 失效信号

材料只提供事实背景，不包含可迁移方法；去掉来源后动作无法成立；提取结果不能指导判断或行动；产出仍像观点摘要。

## 最高风险边界

不要把带有强身份、强时代、强资源条件的做法直接迁移到普通场景；必须剥离条件，否则容易制造伪方法。

## 常组合

- `反权威背书`：提取后去来源化。
- `微技能封装`：把动作封装成可调用技能。
- `问题分型`：判断动作属于哪类问题。
- `完成检查`：检查提取结果是否可用。
- `事实-推断-假设分离`：避免把解释当事实。
