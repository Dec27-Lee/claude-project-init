# 承接责任检查

> 动作类型：关系承接
> 场景标签：关系、个人、组织、生活、工作、AI、平台、政策、个人成长

## 适用处境

个人、团队或组织把希望寄托在外部强者、领导、平台、贵人、工具、AI、政策上，但尚未明确自己必须承接什么动作。

## 不适用处境

当事人处于紧急伤害、胁迫、基本保障缺失、未成年人或无承接能力处境；外部责任本就不可转嫁；当前首先需要救急或安全保护。

## 输入

- 外部帮助对象和可提供资源。
- 自己想要的结果、当前能力和失败后果。
- 必须由自己完成的学习、执行、质量、维护、复盘动作。
- 是否存在权力不对等、经济依赖或制度性责任。

## 操作步骤

1. 写清期待外部提供什么：资源、入口、工具、机会、训练、反馈、背书或保护。
2. 写清最终想获得的结果，以及谁会承担结果失败的后果。
3. 拆出自己必须承接的动作：学习、执行、质量、维护、风险、复盘和长期改进。
4. 判断外部帮助是否会变成依赖、幻想、道德绑架或责任转嫁。
5. 把“帮我得到结果”改写成“帮我建立能拿结果的结构”。
6. 如果自己无承接能力或外部责任不可转嫁，停止自责式分析，切换到保护或责任边界。

## 关键自问

- 如果外部资源明天消失，我还能推进哪一步？
- 我是在要入口、工具、训练、反馈，还是在要别人替我承担结果？
- 哪些责任必须由平台、组织、制度或专业方承担？
- 我最小可自承接动作是什么？
- 是否有人借“自我承接”把责任推给更弱的一方？

## 输出格式

```markdown
- 外部可提供：
- 外部不可替代：
- 自己必须承接：
- 不可转嫁责任：
- 依赖风险：
- 下一步自承接动作：
```

## 常见误用

- 把系统性问题简化为个人不努力。
- 用“自我承接”替代组织、制度、法定或安全责任。
- 要求能力不足的一方一次性承担完整后果。
- 把外部帮助全部否定，忽略训练、反馈和资源入口的价值。

## 失效信号

救急、生命安全、基本保障、制度性不公、权力不对等、胁迫控制、亲密暴力、职场霸凌、经济依赖、未成年人或无承接能力对象，不能用本技能要求弱势方自承接或推卸外部责任；切换到安全保护、救急最小行动或责任边界判断。

## 最高风险边界

不能把外部应承担的责任转嫁给缺乏选择权、知情权、资源或承接能力的一方。

## 常组合

- `最小行动`：把可自承接部分落到第一步。
- `承接链结构设计`：需要长期帮助而非一次性建议时。
- `规律代价对齐检查`：判断目标所需代价是否能承接。
- `朋友请求边界问询`：外部帮助混入关系压力时。
