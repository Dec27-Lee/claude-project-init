# 方法论版本化复盘

> 动作类型：方法建设
> 场景标签：复盘、组织、方法建设、长期项目、工作、生活

## 适用处境

过去有效的方法、原则、流程或口号在新阶段开始失灵；团队仍在机械复用旧经验，或把阶段性成功经验当作永恒信条。需要判断哪些保留、哪些升级、哪些停用。

## 不适用处境

旧方法只是执行不到位，还没有证据表明问题结构变化；当前需要的是事实查证、救急止损或低风险执行；方法本身涉及专业法律、医疗、财务、安全等领域时，不做业余版本化替代专业规范。

## 输入

- 旧方法/原则/流程的原始适用场景和成功条件。
- 当前阶段变化：规模、用户、技术、组织、竞争、规则、资源、风险。
- 失败案例、偏差案例、仍然有效的案例和不可变底线。

## 操作步骤

1. 写出旧方法解决的原问题：当时面对什么约束、目标和环境。
2. 对比当前问题结构：用户、规模、资源、组织、技术、渠道、规则是否发生变化。
3. 拆分旧方法：保留的底层判断、需要升级的操作、必须停用的旧动作。
4. 找失效信号：旧方法继续使用会带来什么误判、成本、风险或组织惯性。
5. 设计新版本：新触发条件、新步骤、新输出、新失效信号和组合对象。
6. 设置复查点：用真实场景验证新版方法，而不是写成更漂亮的口号。

## 关键自问

- 这个方法当初为什么有效？那些条件现在还在吗？
- 我们是在坚持原则，还是在复制旧动作？
- 旧方法现在制造了什么新的副作用？
- 哪些底层判断仍成立，哪些执行方式必须改？
- 新版本能否导出更清晰的行动和停止条件？

## 输出格式

```markdown

- 原适用场景：

- 输出：

```

## 常见误用

- 因为遇到困难就否定旧方法，而不是区分执行问题和结构变化。
- 把成功经验包装成永恒原则。
- 只改名字，不改触发条件、动作和失效信号。
- 为迎合新叙事频繁升级方法，导致组织无所适从。
- 忽略不可变底线，把原则降级成权宜之计。

## 失效信号

没有证据表明问题结构变化；复盘只是在争口号；新版方法无法导出行动；专业规范被业余方法替代；组织需要稳定执行而非再造方法。命中时切换到 `完成检查`、`主要矛盾` 或专业支持。

## 最高风险边界

不能用“方法升级”随意推翻安全、合规、伦理、财务和用户承诺底线；不能让方法论成为组织反复折腾的借口。

## 常组合

- `判断动作提取`：从案例、记录或经验中提取可升级动作。
- `微技能封装`：把新版本封装成可调用技能。
- `反权威背书`：避免依赖来源或成功案例背书。
- `完成检查`：验证新版方法是否能独立运行。
- `阶段轮廓与转折点管理`：判断是否确实进入新阶段。
