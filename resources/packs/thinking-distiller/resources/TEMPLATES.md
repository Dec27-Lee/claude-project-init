# thinking-distiller 模板

用于把材料蒸馏为 `clear-thinking` 可调用微技能的标准模板。

---

## 0. Session 目录初始化模板

每次正式蒸馏建议在 `local/thinking-distiller/sessions/` 下创建一个独立过程目录：

```text
local/thinking-distiller/sessions/YYYY-MM-DD-短标题/
├── brief.md
├── material-map.md
├── extraction-cards.md
├── candidate-skills.md
├── precision-review.md
├── candidate-skills-v2.md        # 有重大修正时建议
├── integration-plan.md
└── tests.md
```

并在 `local/thinking-distiller/index.md` 增加一行：

```markdown
| YYYY-MM-DD | 标题 | in_progress | 材料/人物 | 目标动作类型/场景标签/微技能 | `sessions/YYYY-MM-DD-短标题/` | 待接入 | 备注 |
```

## 3. 判断动作提取卡

```markdown
## 判断动作提取卡：<动作候选名>

- 材料位置：
- 来源主体：谁做出了判断动作？
- 辅助主体：谁提供了样本、资源、限制条件或执行建议？
- 主体边界：这是某个人稳定方法，还是多人协作结果？
- 原始场景：材料中人物/作者面对的问题是什么？
- 原文明确支持：原文中明确发生的行动、安排、取舍、反馈是什么？
- 蒸馏者推断：哪些是根据上下文补出的解释？
- 关键信号：他先看到了什么事实、模式或异常？
- 排除噪声：他没有被什么带偏？
- 动作类型：风险识别 / 边界设置 / 结构设计 / 资源定位 / 验证机制 / 体验临摹 / 其他
- 判断动作：他实际做了什么思考动作？
- 结构设计要素：参与方、权责关系、激励/约束、承接机制、反馈闭环；非结构设计可写“不适用”。
- 取舍方式：他如何决定选/不选什么？
- 验证方式：他如何验证、等待反馈或修正？
- 可迁移处境：用户在什么工作/生活问题中会遇到类似结构？
- 失效场景：什么时候不能用？
- 是否可能拆分：否 / 是，拆成哪些动作？
- 最危险误用：
- 初步结论：保留 / 合并 / 拆分 / 修订 / 暂缓 / 放弃
```

## 6. clear-thinking 运行层微技能格式

最终写入 `micro-skills/<微技能>.md` 独立文件时使用。

```markdown
---
name: <微技能名>
action_type: <动作类型>
scene_tags: [工作, 生活, 产品]
output: <该技能产生的中间结论>
---

# <微技能名>

> 动作类型：<动作类型>
> 场景标签：工作、生活、产品

## 适用处境

什么问题形态下调用；写“用户卡住的状态”，不要只写关键词。

## 不适用处境

哪些情况不要调用，尤其是高风险、专业边界、低风险直接执行或事实不足场景。

## 输入

需要哪些事实、材料、角色、约束和待确认假设。

## 操作步骤

1. 第一步判断什么。
2. 第二步拆什么变量。
3. 第三步形成什么中间结论。
4. 必要时说明何时切换到其他微技能。

## 关键自问

- 本技能要回答的关键判断问题是什么？
- 哪个条件不满足时判断会失效？
- 是否存在权力不对等、合规或安全边界？

## 输出格式

- 当前判断：
- 关键依据：
- 待确认项：
- 下一步：
- 失败/切换信号：

## 常见误用

- 不要把本技能用成什么。

## 失效信号

何时不用、停止、追问、查证、切换技能或交给专业支持。

## 最高风险边界

高风险或易误用微技能必填；低风险可省略本节或写“不适用”。

## 常组合

- `<已有微技能>`：组合原因。
```

注意：运行层不写“来源材料”“某人说过”“书中提到”。新增/修改运行层微技能时，默认保持上述完整小节；低风险轻量技能可压缩正文但不要回退成短骨架。高风险微技能必须补充“最高风险边界”，明确绝不能用本技能做什么。
